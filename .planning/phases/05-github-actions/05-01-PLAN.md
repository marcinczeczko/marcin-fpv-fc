<objective>
Automate the generation of all KiCad project outputs using GitHub Actions.

Purpose: Ensure every push to the repository results in a verified, up-to-date set of manufacturing and documentation artifacts.
Output: Integrated CI/CD pipeline.
</objective>

<execution_context>
The project uses KiBot v2_k9 for KiCad 9 compatibility.
Modular configurations are stored in kibot_yaml/.
</execution_context>

<tasks>

<task type="auto">
  <name>Task 1: Create GitHub Actions Workflow</name>
  <files>.github/workflows/kibot.yml</files>
  <action>
Create the workflow file `.github/workflows/kibot.yml` with the following configuration:
- Trigger on `push` to `main` and all `pull_request`.
- Single job `kibot` running on `ubuntu-latest`.
- Use `INTI-CMNB/KiBot@v2_k9`.
- Target the `RELEASED` group (or pass `VARIANT=RELEASED` via environment if using the group-based system).
- Use `actions/upload-artifact@v4` to upload the generated directories.

Workflow contents:
```yaml
name: KiBot Output Generation

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  kibot:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Generate Outputs (RELEASED)
        uses: INTI-CMNB/KiBot@v2_k9
        with:
          config: polski_fc.kibot.yaml
          # We use groups, so we pass the group name as a target
          targets: RELEASED
          # We also pass VARIANT=RELEASED to ensure text variables are set correctly
          variant: RELEASED

      - name: Upload Schematic
        uses: actions/upload-artifact@v4
        with:
          name: Schematic
          path: Schematic/

      - name: Upload Manufacturing
        uses: actions/upload-artifact@v4
        with:
          name: Manufacturing
          path: Manufacturing/

      - name: Upload 3D
        uses: actions/upload-artifact@v4
        with:
          name: 3D-Model
          path: 3D/
```
  </action>

  </verify>
  <done>Workflow file created and configured.</done>
</task>
